# Image Lab — редактор изображений с поддержкой GrayBit-7

Веб-редактор растровых изображений: загрузка и сохранение PNG / JPEG и
собственного учебного формата **GrayBit-7 (`.gb7`)**, просмотр по каналам,
пипетка, градационная коррекция «Уровни», масштабирование с собственной
интерполяцией и фильтрация свёрткой. Вся обработка изображений реализована
вручную, без сторонних графических библиотек.

**Деплой:** https://marvelous-mermaid-b5af86.netlify.app/

## Стек

- **React 18** + **Vite 5** (JSX, ES-модули);
- HTML5 **Canvas** для отрисовки;
- **Web Worker** для тяжёлой свёртки;
- без внешних зависимостей для обработки изображений — всё своё.

## Запуск

```bash
npm install
npm run dev      # дев-сервер (http://localhost:5173)
npm run build    # продакшен-сборка в dist/
npm run preview  # локальный просмотр собранной версии
```

## Возможности

### 1. Загрузка, отображение и сохранение

- Открытие файлов **PNG, JPG/JPEG, GB7** — кнопкой «Открыть» или
  перетаскиванием в окно (drag-and-drop).
- Отрисовка на HTML5 `<canvas>`.
- Сохранение в **PNG, JPG, GB7** (кодер GB7 переводит изображение в оттенки
  серого; прозрачность сохраняется как 1-битная маска).
- Строка состояния: ширина, высота, **глубина цвета**, формат, режим каналов,
  активный инструмент и данные выбранного пикселя.
- Декодер и кодер GrayBit-7 написаны вручную ([`src/lib/gb7.js`](src/lib/gb7.js)).

### 2. Каналы и пипетка

- **Панель каналов** с миниатюрами: изображение раскладывается на независимые
  каналы (Gray / RGB / Alpha в зависимости от типа: 1, 2, 3 или 4 канала).
  RGB-каналы и альфа показаны в градациях серого (белый — максимум интенсивности).
- Клик по каналу включает/выключает его отображение на главном холсте;
  состояние наглядно подсвечено. Если оставлен только альфа-канал — виден
  чёрно-белый снимок маски прозрачности.
- **Пипетка** (инструмент в панели): по клику показывает координаты пикселя
  относительно изображения, значения **RGB (0–255)** и координаты в
  пространстве **CIELAB**. Координаты курсора корректно пересчитываются с учётом
  текущего масштаба холста.

### 3. Уровни (Levels) — градационная коррекция

Инструмент работает в модальном окне (`<dialog>`).

- **Гистограмма** тонового диапазона: композитная (по светлоте `0.299R +
  0.587G + 0.114B`) либо по выбранному каналу; переключение **линейной /
  логарифмической** шкалы.
- **Входные уровни (Input Levels):** три маркера — точка чёрного, гамма
  (полутона), точка белого. Гамма 0.1–9.9 (степенная коррекция: влево —
  осветление, вправо — затемнение). Чёрная точка не заходит за белую,
  маркер гаммы всегда между ними. Есть и числовые поля для точного ввода.
- Коррекция применяется к **Master (все RGB)** или к **отдельному каналу,
  включая Alpha**; настройки сохраняются при переключении каналов.
- **Предпросмотр** в реальном времени (чекбокс, по умолчанию включён); его
  снятие мгновенно возвращает оригинал для сравнения «до/после».
- Кнопки **Сбросить** (не закрывает окно), **Отмена** (закрывает + откат),
  **Применить**. Под капотом — таблицы подстановки (LUT) и троттлинг через
  `requestAnimationFrame`.

### 4. Масштабирование

- **Собственная интерполяция**: метод ближайшего соседа и билинейная
  ([`src/lib/interpolation.js`](src/lib/interpolation.js)). По умолчанию —
  билинейная. Реестр методов расширяемый.
- Главный холст масштабируется именно этими реализациями (а не CSS).
- **Range-слайдер 12–300%** в панели; изображение перерисовывается по центру.
- При загрузке изображение автоматически вписывается в холст с отступом ≥50px,
  масштаб ограничен диапазоном 12–300%.
- **Инструмент «Размер изображения»** (модальное окно): количество пикселей
  до и после в мегапикселях, выбор единиц (проценты / пиксели), поля ширины и
  высоты, чекбокс связи пропорций, выбор алгоритма интерполяции с тултипом-
  описанием и валидацией ввода.

### 5. Фильтрация (свёртка)

Инструмент в модальном окне, аналог Photoshop *Filters → Custom*.

- Сетка **3×3** для ввода ядра и список **преднастроек**: тождественное,
  повышение резкости, фильтр Гаусса 3×3, прямоугольное размытие и два ядра
  оператора **Прюитта**. Значения пресета можно править вручную.
- Выбор **каналов**, к которым применяется фильтр (все или отдельные).
- **Стратегии обработки краёв** (padding): копирование края, заполнение чёрным,
  заполнение белым — размерность изображения не уменьшается.
- **Предпросмотр** и кнопки сброс / отмена / применить.
- Свёртка считается в **Web Worker** ([`src/workers/filter.worker.js`](src/workers/filter.worker.js)),
  поэтому интерфейс не блокируется на больших изображениях.

## Формат GrayBit-7

Учебный формат изображений в оттенках серого без сжатия: 7 бит на пиксель для
яркости, 8-й бит — опциональная маска прозрачности.

| Смещение | Размер | Поле |
|---|---|---|
| 0 | 4 | Сигнатура `47 42 37 1D` («GB7·») |
| 4 | 1 | Версия (`0x01`) |
| 5 | 1 | Флаги (бит 0 — наличие маски) |
| 6 | 2 | Ширина (uint16, big-endian) |
| 8 | 2 | Высота (uint16, big-endian) |
| 10 | 2 | Резерв (`0x0000`) |
| 12 | W×H | Данные: биты 6–0 — яркость, бит 7 — маска |

Реализация — [`src/lib/gb7.js`](src/lib/gb7.js).

## Архитектура

Логика обработки изображений вынесена из UI в чистые модули (без React), что
делает её тестируемой и переиспользуемой.

```
src/
├── lib/                  чистая логика (без DOM/React)
│   ├── gb7.js            кодер/декодер GrayBit-7
│   ├── color.js          RGB → CIELAB
│   ├── image.js          каналы, характеристики, композит, выборка пикселя
│   ├── levels.js         LUT, гистограмма, гамма
│   ├── interpolation.js  ближайший сосед, билинейная (расширяемый реестр)
│   ├── convolution.js    свёртка 3×3, пресеты ядер, обработка краёв
│   └── fileio.js         определение формата, загрузка/сохранение
├── hooks/
│   ├── useImageDocument.js  жизненный цикл файла, статус
│   ├── useImageView.js      каналы, зум, инструмент, предпросмотр
│   └── useFilterWorker.js   обёртка над Web Worker свёртки
├── components/
│   ├── Toolbar, Sidebar, CanvasStage, StatusBar
│   ├── Modal              обобщённое окно на <dialog>
│   ├── ChannelPanel, ChannelThumbnail, Histogram, LevelsSlider
│   └── LevelsDialog, ResizeDialog, FilterDialog
├── workers/
│   └── filter.worker.js  свёртка в фоновом потоке
└── App.jsx               композиция
```

Принципы: исходные пиксели изображения **никогда не мутируются** — для вывода
и предпросмотра создаются новые буферы; горячие пути (композит, ресэмплинг,
свёртка) оптимизированы под минимум аллокаций.
